## Firestore Question Document — Exact Field Names
**Critical reference — AI Studio always gets these wrong:**
```
answer        "Gluing neighboring cells"   ← correct answer (NOT correct_answer)
createdAt     "2026-03-27T10:11:25.655Z"
examName      "NEET-UG"                    ← exam context (NOT subject)
explanation   "CONCEPT:...APPLICATION:...NCERT:...EXAM TIP:..."
id            "bio_n_27.03.2026_15.37_63"  ← subject encoded in id prefix
level         "Easy"                       ← difficulty (some docs only)
options       ["opt1","opt2","opt3","opt4"]
question      "Which process describes..."
topic         "Passive Transport"          ← topic name (NOT type)
trick         "..."                        ← alternate explanation (some docs only)
type          "..."                        ← alternate topic (some docs only)
```

**Subject detection — from id prefix:**
- `bio_` → Biology
- `phy_` → Physics
- `chem_` → Chemistry

**Field aliases across collections:**
- `answer` = correct answer (always present)
- `examName` OR `level` = exam/difficulty context
- `topic` OR `type` = topic category
- `explanation` OR `trick` = explanation text

## AV Explainer — Final Design

### Layout
- Centered NISH-LOGIC logo + title at top
- Teacher (left) and Student (right) — podcast style
- Question card below characters — shows current wrong question being discussed
- Transcript panel below question card
- Controls at bottom: Speed, Play/Pause, Restart, EN/HI toggle, counter

### Script Logic (script-gen.js)
- Generic — works for any app, reads subject names from breakdown keys
- Discusses 50% of wrong questions minimum
- For each wrong question: answer mismatch + CONCEPT from explanation + EXAM TIP
- EN mode: English text + en-IN voice
- HI mode: Hindi Devanagari text + hi-IN voice
- MCQ explanation (CONCEPT/EXAM TIP) always in English regardless of toggle — natural code-switch
